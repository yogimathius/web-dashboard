# Security Policy

## Supported Versions
- Active development happens on the default branch.
- Security fixes are generally applied to the default branch first.

## Reporting a Vulnerability
- Please do not open a public issue for security vulnerabilities.
- Use GitHub private vulnerability reporting (Security Advisories) for this repository.
- If private reporting is unavailable, open an issue without exploit details and request a secure contact channel.

## Response Expectations
- Initial triage target: within 5 business days.
- If accepted, fix timeline depends on severity and maintenance status.
